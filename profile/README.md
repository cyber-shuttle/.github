## Cybershuttle - Unified Research Across Local, Cloud, and HPC

Cybershuttle effortlessly connects your laptop, cloud resources, and supercomputers into one streamlined research environment. Run quick analyses locally, seamlessly offload intensive simulations to remote HPC clusters, and have your data automatically flow where you need it. No rewriting code. No manual transfers. Just smoother, faster science.

By intelligently optimizing task placement and data movement, Cybershuttle accelerates computational workflows beyond what purely local or remote solutions can achieve.

Powered by [Apache Airavata](https://github.com/apache/airavata), Cybershuttle empowers researchers with a unified, friction-free computing experience.

---

### Core Ecosystem

The infrastructure that powers Cybershuttle, tracked on the [project board](https://github.com/orgs/cyber-shuttle/projects/2).

| Component | Description |
|-----------|-------------|
| **CS-Bridge** | VS Code extension for managing SSH sessions, HPC jobs, dev tunnels, and remote storage |
| [**linkspan**](https://github.com/cyber-shuttle/linkspan) | Tunnel and mount agent — handles secure connections between local and remote environments |
| [**frp-server**](https://github.com/cyber-shuttle/frp-server) | FRP relay server with REST API, SQLite, and JWT auth for tunnel management |
| **vscode-ux-metrics** | UX metrics collection for research session telemetry |
| **cs-bridge-admin** | Admin server for CS-Bridge fleet management and metrics reporting |
| [**mcp-server**](https://github.com/cyber-shuttle/mcp-server) | MCP Server for Cybershuttle AI integrations |
